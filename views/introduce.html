2018. 05 ~ 2018. 09
웹 서비스 개발
Html, CSS, Javascript, Angularjs
선박을 모니터링하고 성능을 분석해주는 다양한 서비스가 있는 웹 프로젝트입니다. VESSELLINK FOR SHIP 프로그램에 들어가게 될 커스터마이징 템플릿 파일을 추출하는 페이지 개발을 담당하였습니다. 개발하면서 Angularjs에서 제공하는 2-Way Binding 서비스를 남발하여 페이지 성능저하 이슈가 있었습니다. 양방향 데이터 바인딩의 장점만 보고 사용하여 발생한 문제였고 이후, 양방향 데이터 바인딩의 문제점을 인식하고 단방향 바인딩으로 변경하면서 성능을 향상시킬 수 있었습니다.
느낀점
웹 개발자의 꿈을 안고 처음으로 개발하게 된 프로젝트 입니다. 그 당시에 react, vue, angular(ver 2.0 이상)의 프레임워크들이 엄청나게 떠오르던 시절이었습니다. 회사에서 Deprecated된 Angularjs를 사용하고 있어 난해하였지만, Angular2 프레임워크를 학창시절에 사용한 경험을 기반으로 빠르게 적응할 수 있었습니다. 프로젝트 초창기에 숙련된 웹 개발자가 없어서 인터넷에 무료 템플릿 페이지를 기반으로 프로젝트가 구축되어있었습니다. 또한, 사용되는 라이브러리나 리소스들은 필요한 것 이외에도 모든게 들어가 있었고, 프로젝트가 관리가 되고있지 않았습니다. 이후에 회의를 주도하여 프론트엔드 개발자 동료들과 프로젝트 구조에 대해서 많은 회의를 주도하여 문제점을 인식한 후 구조를 개선하고, 코드 관리 포인트를 추가해 나가면서 웹 개발의 첫 발걸음을 내딛게 된 프로젝트였습니다.

느낀점
